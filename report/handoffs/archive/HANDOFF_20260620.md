## The two diagnosed gaps (these are the work, and they're real)

Both are deck-specific *tech*, not core-logic bugs. Building them is the card
domain model, not copying anyone's policy.

1. **No gust.** `CardData` has no effect text, so Boss's Orders / gust can't be
   detected generically → the plan never targets a benched Pokémon (`tgt=0`
   always). The 1084's KO-the-support-then-sweep line is unavailable to us.
2. **Crustle wall unsolved (THE lever).** Crustle (345) nullifies Pokémon-ex
   damage. Our `wall_immune` skips ex-into-Crustle but doesn't route energy to a
   non-ex backup attacker (Hariyama line) and switch it active → vs Crustle we
   have no viable attacker and lose 0%. Solving this one matchup is the proven
   0.47→0.77 swing that defines the 600→1084 jump.

## What's next (recommended, in order)

1. **Add a thin per-deck "tech" layer over `RuleCoreScorer`** — a small table per
   deck of: gust card ids (so the plan can target the bench), the non-ex backup
   attacker id + its energy needs (so we beat Crustle), and energy-accel ability
   ids. Re-gate after each addition; ship only when suite clears ~50% and the
   Crustle matchups are no longer 0%. This is the fastest path to actually climbing.
2. **Broaden opponent detection** beyond crustle/water: psychic-control and
   fast-tempo families (meta moves day-to-day — `data/kaggle_ref/community/
   daily-public-meta-notes...`). Don't overfit Crustle.
3. **Deck quality:** start from the 1084's *tuned* Lucario list (extra Boss's
   Orders / Riolu / Gravity Mountain) rather than vanilla `real_mega_lucario_ex.csv`.
4. **Then exceed the public ceiling with search:** `rl/az_train.py` proves MCTS
   runs at ~15ms/move; use the strong rule core as the prior. This is the
   leaders' (1300) edge over the 1084 rule agent.
